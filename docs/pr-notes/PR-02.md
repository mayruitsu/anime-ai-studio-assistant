# PR #02 FastAPI最小構成を実装

## 概要

DockerコンテナでFastAPIサーバーを動かす最小構成を実装した。`GET /health` にアクセスすると `{"status": "ok"}` が返る。

## 使用ライブラリ

| ライブラリ | 役割 |
|---|---|
| fastapi | WebフレームワークでAPIのルーティングを定義する |
| uvicorn | FastAPIを動かすASGI対応のWebサーバー |

## 処理の流れ

1. `docker compose up --build` でDockerイメージをビルド・起動
2. uvicornが `main.py` を読み込みサーバーを立ち上げる
3. `http://localhost:8080/health` にアクセスするとレスポンスが返る

## 重要なコード解説

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/health")
def health():
    return {"status": "ok"}
```

- `FastAPI()` : アプリケーションのインスタンスを作成する
- `@app.get("/health")` : `GET /health` へのリクエストをこの関数に紐づけるデコレータ
- 辞書を `return` すると FastAPI が自動でJSONに変換してくれる

## 学習ポイント

- FastAPIはPythonの型ヒントを活用した軽量なWebフレームワーク
- `uvicorn main:app` の `main` はファイル名、`app` は変数名を指す
- Dockerfileの `CMD` に起動コマンドを書くことでコンテナ起動時に自動実行される
