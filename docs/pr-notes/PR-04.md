# PR #04 差分チェックをコードとドキュメントで分けて表示

## 概要

PRの差分チェックで、コードとドキュメントの行数を分けて表示するよう改善した。100行超えのエラー判定はコードのみを対象とし、ドキュメントは除外される。

## 変更ファイル

- `.github/workflows/pr-check.yml`

## 処理の流れ

1. 全ファイルの差分行数を `TOTAL` として取得
2. `docs/` を除いたコードのみの差分行数を `CODE` として取得
3. ログに「合計 / コード / ドキュメント」の内訳を表示
4. `CODE` が100行を超えた場合のみエラーにする

## 重要なコード解説

```bash
TOTAL=$(git diff --shortstat origin/main...HEAD | awk '{print $4+$6}')
CODE=$(git diff --shortstat origin/main...HEAD -- ':!docs/' | awk '{print $4+$6}')
echo "差分行数: 合計 ${TOTAL:-0}行（コード ${CODE:-0}行 / ドキュメント $((${TOTAL:-0} - ${CODE:-0}))行）"
```

- `--shortstat` : 追加・削除行数を1行で出力するオプション
- `awk '{print $4+$6}'` : 追加行数と削除行数を足して合計を出す
- `${VAR:-0}` : 変数が空のとき `0` をデフォルト値として使う記法
- `$((A - B))` : bash の算術演算で差分を計算

## 学習ポイント

- Git pathspec の `:!パターン` で特定ディレクトリを除外できる
- `awk` は出力の特定列を抽出・計算するのに使う
- bash の `${VAR:-デフォルト}` は変数が未定義・空のときの安全策として有用
