# PR #05 React環境構築（Vite + Docker）

## 概要

フロントエンドのReact環境をViteで構築し、Dockerで起動できるようにした。`http://localhost:5173` でReactアプリが表示される。

## 追加ファイル

| ファイル | 役割 |
|---|---|
| `frontend/Dockerfile` | フロントエンド用のDockerイメージ定義 |
| `frontend/package.json` | 使用するnpmパッケージの定義 |
| `frontend/vite.config.js` | Viteの設定ファイル |
| `frontend/index.html` | ReactアプリのエントリーポイントとなるHTML |
| `frontend/src/main.jsx` | ReactをDOMにマウントする起点 |
| `frontend/src/App.jsx` | アプリのルートコンポーネント |

## 使用ライブラリ

| ライブラリ | 役割 |
|---|---|
| react / react-dom | UIを構築するフレームワーク |
| vite | 高速な開発サーバー・ビルドツール |
| @vitejs/plugin-react | ViteでReactを使うためのプラグイン |

## 処理の流れ

1. `docker compose up --build` でfrontendサービスが起動
2. Viteが `index.html` を配信
3. `index.html` が `src/main.jsx` を読み込む
4. `main.jsx` が `App.jsx` をDOMの `#root` にマウント
5. ブラウザに「アニメーションアシスタントAI」が表示される

## 重要なコード解説

```jsx
// main.jsx
createRoot(document.getElementById("root")).render(
  <StrictMode>
    <App />
  </StrictMode>
);
```

- `createRoot` : ReactをDOMの特定の要素に紐づける関数
- `document.getElementById("root")` : `index.html` の `<div id="root">` を取得
- `<App />` : コンポーネントをJSX記法で呼び出す書き方

```yaml
# docker-compose.yml
volumes:
  - ./frontend:/app
  - /app/node_modules
```

- `- /app/node_modules` : ホストのファイルでnode_modulesが上書きされないようにする設定

## 学習ポイント

- Viteは従来のCreate React Appより高速な開発サーバーとして現在主流
- JSXはJavaScriptの中にHTMLライクな記法を書ける構文（Reactの特徴）
- `--host` オプションをつけることでDockerコンテナ外からアクセスできるようになる
