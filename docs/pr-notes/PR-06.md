# PR #06 画像アップロードUI

## 概要

画像ファイルを選択するとプレビューが表示されるアップロードUIを実装した。選択した画像のURLを親コンポーネントに渡す設計にしており、次のCanvas表示と繋げやすくなっている。

## 変更ファイル

| ファイル | 内容 |
|---|---|
| `frontend/src/components/ImageUploader.jsx` | 画像アップロード・プレビューコンポーネント |
| `frontend/src/App.jsx` | ImageUploaderを組み込み |

## 処理の流れ

1. ユーザーがファイルを選択する
2. `handleChange` が発火し `URL.createObjectURL` で画像のURLを生成
3. `preview` に保存してプレビュー表示
4. `onUpload(url)` で親コンポーネント（App）にURLを渡す

## 重要なコード解説

```jsx
const [preview, setPreview] = useState(null);
```

- `useState` はReactのHooks。コンポーネント内で変化する値（状態）を管理する
- `preview` が現在の値、`setPreview` が値を更新する関数
- 初期値は `null`（まだ画像がない状態）

```jsx
const url = URL.createObjectURL(file);
```

- ブラウザが一時的に画像ファイルに対してURLを発行する仕組み
- サーバーへのアップロードなしにプレビュー表示できる

```jsx
{preview && <img src={preview} ... />}
```

- `&&` は「左が truthy なら右を表示する」という条件付きレンダリング
- `preview` が `null` のときは何も表示しない

```jsx

```

- `{ onUpload }` は親から受け取る props（プロパティ）
- 親が `<ImageUploader onUpload={setImage} />` と書くことで関数を渡している

## 学習ポイント

- `useState` はReactで最もよく使うHooks。値が変わると画面が自動で再描画される
- コンポーネントを分ける理由は「責務を分離するため」。Appは全体管理、ImageUploaderは画像選択だけを担当する
- propsは親から子へ値や関数を渡す仕組み
