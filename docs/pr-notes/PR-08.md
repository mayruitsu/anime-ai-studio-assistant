# PR #08 ポイントをドラッグで動かせる

## 概要

Canvas上の関節ポイントをマウスでドラッグして動かせる機能を実装した。ポイントをstateで管理することで、座標更新のたびにCanvasが再描画される。

## 変更ファイル

| ファイル | 内容 |
|---|---|
| `frontend/src/components/PoseCanvas.jsx` | ドラッグ機能を追加 |

## 処理の流れ

1. マウスを押す（mousedown）→ 近くにポイントがあればインデックスを `dragging` に保存
2. マウスを動かす（mousemove）→ `dragging` がある場合、対象ポイントの座標を更新
3. マウスを離す（mouseup）→ `dragging` をリセット
4. `points` が変わるたびに `useEffect` が発火してCanvasを再描画

## 重要なコード解説

```jsx
const dragging = useRef(null);
```

- ドラッグ中のポイントインデックスを保持する
- `useState` ではなく `useRef` を使う理由：インデックスが変わっても再描画は不要なため

```jsx
const idx = points.findIndex((p) => Math.hypot(p.x - x, p.y - y) < 0.03);
```

- `Math.hypot` で2点間の距離を計算する
- `0.03` は「クリック判定の範囲」。相対座標なので画像サイズに関係なく同じ感度になる

```jsx
setPoints((prev) =>
  prev.map((p, i) => (i === dragging.current ? { ...p, x, y } : p))
);
```

- `prev.map` で配列を新しく作り直す（直接変更しない）
- ドラッグ中のポイントだけ座標を更新し、他はそのまま返す
- `{ ...p, x, y }` はスプレッド構文で `p` の内容をコピーしつつ `x` と `y` だけ上書き

```jsx
const getPos = (e) => {
  const rect = canvasRef.current.getBoundingClientRect();
  return {
    x: (e.clientX - rect.left) / rect.width,
    y: (e.clientY - rect.top) / rect.height,
  };
};
```

- `getBoundingClientRect` でCanvas要素の画面上の位置を取得
- マウス座標からCanvas左上の座標を引いて相対位置を出し、幅・高さで割って0〜1に正規化

## 学習ポイント

- 再描画が必要な値は `useState`、再描画不要な値は `useRef` で管理する
- 配列のstateを更新するときは直接変更せず `map` で新しい配列を作るのがReactの作法
- `Math.hypot(dx, dy)` は三平方の定理で2点間の距離を返す
