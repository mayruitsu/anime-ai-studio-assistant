# PR #09 フレーム追加・切り替え

## 概要

フレームを追加・切り替えできる機能を実装した。各フレームが独立したポイント座標を持ち、フレームを切り替えるとそれぞれの状態が保持される。新しいフレームは直前のフレームのポイント位置を引き継いで追加される。

## 変更ファイル

| ファイル | 内容 |
|---|---|
| `frontend/src/App.jsx` | framesとcurrentFrameのstate管理を追加 |
| `frontend/src/components/FramePanel.jsx` | フレーム一覧・追加ボタンコンポーネント（新規） |
| `frontend/src/components/PoseCanvas.jsx` | pointsをpropsで受け取る形に変更（state→props） |

## 処理の流れ

1. `App` が `frames`（全フレームのポイント配列）と `currentFrame`（選択中のインデックス）を管理
2. `FramePanel` でフレームボタンをクリック → `currentFrame` が切り替わる
3. `PoseCanvas` は `frames[currentFrame]` のポイントを描画
4. ポイントを動かす → `handlePointsChange` で現在のフレームだけ更新
5. `+ 追加` ボタン → 現在のフレームのポイントをコピーして新しいフレームを追加

## 重要なコード解説

```jsx
const [frames, setFrames] = useState([INITIAL_POINTS]);
const [currentFrame, setCurrentFrame] = useState(0);
```

- `frames` は「フレームの配列」で、各要素がポイントの配列
- `currentFrame` は今選択しているフレームの番号

```jsx
const handlePointsChange = (newPoints) => {
  setFrames((prev) =>
    prev.map((f, i) => (i === currentFrame ? newPoints : f))
  );
};
```

- 現在のフレームだけポイントを更新し、他のフレームはそのまま
- PR #08の `points.map` と同じパターンで、今度はフレームの配列に適用している

```jsx
const handleAddFrame = () => {
  setFrames((prev) => [...prev, [...frames[currentFrame]]]);
  setCurrentFrame(frames.length);
};
```

- `[...frames[currentFrame]]` で現在のフレームのポイントをコピーして追加
- `frames.length` は追加前の長さ = 新しいフレームのインデックス

## 学習ポイント

- stateを上位コンポーネント（App）に持ち上げることで複数コンポーネントから共有できる
- `[...array]` はスプレッド構文で配列をコピーする。直接参照を渡すと意図しない共有が起きる
- コンポーネントの責務：Appは状態管理、FramePanelはUI表示、PoseCanvasは描画
