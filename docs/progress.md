# 開発進捗ログ

## 2026-07-20（続き5）

- `rotation.py` を新規作成 — 基準ポーズ（フレーム1）と対象ポーズ（各フレーム）の関節座標から、パーツごとの回転角度（差分）を計算
- `atan2`で各パーツの「付け根→先端」ベクトルの向きを角度に変換し、基準との差分を回転角として返す
- `rotation_test.py` で90度回転させる例を使い、計算結果が期待値と一致することを確認済み
- フェーズ6のタスク「ポーズ→回転角の計算」完了。次はこの角度を使ってパーツ画像を合成するレンダリング処理を進める

## 2026-07-20（続き4）

- `part_cropping.py` を新規作成 — マスクと部位対応表（body_parts.py）を使い、各パーツを回転ピボット付きのRGBA画像チップとして切り出す
- 各パーツのバウンディングボックスは、付け根・先端の関節座標に余白（腕・脚の長さの30%）を加えて算出
- 動作確認用に `part_cropping_test.py` を追加。Docker環境で実写真を使って動作確認し、10パーツ（頭・胴体・上腕×2・前腕×2・太もも×2・すね×2）が期待通り切り出されることを確認済み
- フェーズ6のタスク「パーツ画像の切り出し」完了。次はポーズ→回転角の計算を進める

## 2026-07-20（続き3）

- `body_parts.py` を新規作成 — 関節33点を頭・胴体・上腕×2・前腕×2・太もも×2・すね×2の10パーツにグルーピングする対応表を定義
- 各パーツに回転ピボット（付け根の関節インデックス）を持たせ、次の「ポーズ→回転角の計算」で使えるようにした
- 検出処理自体の追加は不要なため、データ定義のみの軽量な実装
- フェーズ6のタスク「パーツ分割の定義」完了。次はこの対応表とマスクを使ってパーツ画像を切り出す
